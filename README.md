# CS-320 Software Testing and QA

## Overview

This repository contains selected work from CS-320: Software Testing, Automation, and Quality Assurance. The projects included here demonstrate my ability to translate requirements into working software and verify those requirements through structured unit testing. Throughout this course, I focused on writing clean, testable code and applying a strong testing mindset to improve reliability and quality.

## Project One – Contact Service Application

Files included:
- Contact.java  
- ContactService.java  
- ContactTest.java  
- ContactServiceTest.java  

In this project, I developed a contact service application and created comprehensive JUnit tests to validate each requirement. I ensured that constraints such as unique IDs, field length limits, and required fields were enforced through both implementation and automated tests. This project strengthened my understanding of requirement-based testing and reinforced the importance of validating edge cases and boundary conditions.

## Project Two – Testing Strategy and Reflection

Project Two focused on evaluating the overall testing approach used in the application. I analyzed whether my test coverage was sufficient for production release and reflected on how testing supports clean and maintainable code. This project emphasized that testing is not just about confirming functionality, but about uncovering potential defects before deployment.

## Reflection

### How do I ensure my software is functional and secure?

I ensure functionality and security by translating requirements directly into unit tests and validating both expected behavior and edge cases. I use input validation and defensive programming practices to prevent invalid data from entering the system. Writing automated tests alongside development allows me to verify that features behave correctly and reduces the likelihood of defects reaching production.

### How do I interpret user needs and incorporate them into a program?

I begin by breaking down user needs into clear, testable requirements. Each requirement is then implemented as a specific function or constraint within the program, supported by unit tests that confirm correct behavior. By aligning tests with requirements, I ensure the final implementation meets user expectations.

### How do I approach designing software?

I approach software design by identifying responsibilities and separating concerns into modular components. Designing with testing in mind helps keep code maintainable and easier to verify. I consider edge cases early in development and use testing as a tool to guide architectural decisions. My goal is to produce software that is reliable, readable, and structured for long-term maintainability.
